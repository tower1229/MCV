# MCV

A personal AI IDE configuration collection, management, and cross-device deployment tool. MCV captures a developer's AI productivity data into a single private Git repository and deploys it to any new device.

## Language

**Canonical（通用主权层）**:
Configuration content that is semantically shared across multiple AI IDEs — rules, Skills, MCP definitions, workflows. The defining criterion is cross-IDE reusability, not cross-device portability (all three layers are cross-device portable).
_Avoid_: "universal config", "shared config"

**Native（IDE 原生个性层）**:
Configuration that only has meaning within a single specific IDE, stored in its original format. Native config is still cross-device portable via the repository.
_Avoid_: "IDE-specific", "per-IDE"

**Local/Runtime（本机运行状态层）**:
Device-bound data that must never enter the shared repository — credentials, caches, logs, session state, device identifiers.
_Avoid_: "temporary", "transient"

**Repository（数据仓库）**:
A user-owned private Git repository that serves as the single long-term source of truth for all confirmed AI IDE configuration. "Source of truth" means the authoritative human-confirmed version, not the real-time latest state — between captures, the device may hold newer uncommitted changes, and that is by design.
_Avoid_: "sync target", "backup"

**Adapter（适配器）**:
The single per-IDE interface (`IdeAdapter`) that MCV's core uses to detect, capture, and deploy configuration for a specific IDE. Each Adapter internally composes a CanonicalTransformer and a NativeFileHandler.
_Avoid_: "plugin", "driver", "connector"

**CanonicalTransformer**:
The internal component of an Adapter responsible for semantic conversion between MCV's unified Canonical format and a specific IDE's format (e.g. unified rules → `CLAUDE.md`).
_Avoid_: "converter", "mapper"

**NativeFileHandler**:
The internal component of an Adapter responsible for path discovery, file read/write, secret filtering, and variable substitution — without understanding the semantic meaning of configuration fields.
_Avoid_: "file copier", "sync engine"

**Overlay（字段级合并）**:
The mechanism for handling config files that contain both MCV-managed and IDE-native fields. Uses a managed-whitelist strategy: only `managedPaths` are explicitly declared; every undeclared field defaults to Native ownership and is preserved untouched. A small `localPaths` list excludes known device-bound fields.
_Avoid_: "merge strategy", "field mapping"

**Drift（漂移）**:
A local modification to a field that MCV considers managed (i.e. in `managedPaths`). Capture reports drift separately from normal native changes, because drifted fields were generated by MCV and should not be silently absorbed back as native config.
_Avoid_: "conflict", "divergence"

**Capture（收集）**:
The reverse operation that collects device-side configuration changes back into the Repository. Secret sanitization and path parameterization happen automatically as convention-based processing before the user sees any output. The user confirms the processed final form (what will actually be written to the repository), not the raw original values. Summary counts (e.g. "3 secrets excluded, 5 paths parameterized") are shown; per-field transformation details are available via `--verbose`.
_Avoid_: "sync back", "import"

**Canonical Rules（通用规则）**:
A single Markdown file (`common/AGENTS.md`) containing the user's cross-IDE development rules. Named `AGENTS.md` to match mainstream AI IDE convention. The CanonicalTransformer renames/places it per IDE target (e.g. `CLAUDE.md` for Claude Code). Multi-file rule sets are deferred to future versions.
_Avoid_: "rule templates", "rule library"

**MCP Registry（MCP 注册表）**:
A Canonical YAML file (`common/mcp.yaml`) that stores the core definitions of MCP servers (name, command, args, env variable references). IDE-specific overrides (e.g. Cursor's `alwaysAllow`, alternate launch commands) live in `ide/<ide-name>/` and are merged on top during deploy. Override priority: IDE-level > Canonical base.
_Avoid_: "MCP config", "server list"

**Skill**:
A reusable instruction package stored as a directory containing a `SKILL.md` file (with YAML frontmatter) and optional subdirectories (`scripts/`, `examples/`, `resources/`, `references/`). Skills live in `common/skills/` as Canonical content. Deployment is a directory copy to each IDE's global skills path — no content transformation needed. IDEs that don't support Skills are skipped with a notice.
_Avoid_: "recipe", "template", "rule pack"

**Baseline Snapshot（基线快照）**:
A record of file hashes written to local device state after `init` or `deploy`, marking "these files are in sync with the repository at this moment." `mcv init` records a baseline at the end (a virtual deploy — no files are written, but hashes are captured). This gives subsequent `capture` runs a reference point to detect changes.
_Avoid_: "deploy record", "sync state"

**mcv.yaml（仓库清单）**:
The single root-level file that identifies a directory as an MCV Repository and holds all repository-level configuration: identity (`repositoryId`, `schemaVersion`, `initializedAt`), target IDE declarations, variable definitions, security policy, and capture/deploy settings. Replaces the earlier `.mcv/repository.json` — no hidden metadata directory needed.
_Avoid_: "config file", "settings file"

**Secret Scanning（密钥扫描）**:
V0.1 uses two layers: (1) file-name blacklist — known sensitive files like `credentials.json`, `.env`, `id_rsa`, `*.pem` are excluded entirely; (2) field-name pattern matching — JSON/YAML keys containing `secret`, `token`, `key`, `password`, or `credential` have their values replaced with `${env:FIELD_NAME}` references. No value-content regex or external tool integration in v0.1. Capture summary shows "N sensitive fields excluded" count.
_Avoid_: "secret scanner", "credential vault"

**Platform Override（平台覆盖）**:
Platform differences are primarily handled by the variable system (`${HOME}`, `${PROJECTS_HOME}`, etc.). The `overrides/macos/` and `overrides/windows/` directories exist for truly platform-exclusive content (e.g. a Windows-only MCP server). When used, override files replace the corresponding base file entirely (file-level override) — no deep merge in v0.1.
_Avoid_: "platform layer", "OS config"
